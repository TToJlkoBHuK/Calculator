Данный проект представляет собой реализацию Паттерна "Команда"

Основная задача — реализовать механизм инкапсуляции методов класса произвольной сигнатуры в унифицированный интерфейс команды, как это требовалось в задании. Это позволяет вызывать любые методы (```Subject::f3```, ```Subject::f_zero```, ```Subject::f_set_val```) через один общий интерфейс ```ICommand::execute```, передавая аргументы в виде словаря (```std::map<std::string, int>```)

Архитектура:
1) ```ICommand``` - Абстрактный базовый класс, определяющий унифицированный метод ```execute```
2) ```Wrapper<...>``` - Шаблонный класс, который инкапсулирует конкретный метод-член (```&Subject::f3```) и указатель на объект (```subj```). Использует ```std::index_sequence``` для распаковки аргументов из ```std::map``` перед вызовом метода
3) ```Engine``` - Хранит команды по их строковому имени (```"command1"```). Отвечает за вызов команды, не зная о её конкретной реализации
4) ```Subject``` - Класс, содержащий бизнес-логику

Структура:
```
->CMakeLists.txt
->Public
├─> ICommand.h
├─> Wrapper.h
├─> Engine.h
└─> Subject.h
->Private
├─> Engine.cpp
└─> main.cpp
```
Сборка:
1) ```mkdir build```<br>
   ```cd build```
2) ```cmake .. -G "Ninja"```
3) ```cmake --build .```
4) ```pattern_command.exe```
