Приложение состоит из исполняемого файла - .exe, который имеет консольный интерфейс и способен вычислять выражения с операциями ```+ - / * ()```.

Ввод: ```16 + 4 * (3 – 1)```

Вывод: ```24```

При запуске приложение обращается в папку ```./plugins```, и из каждой dll загружает и распознает соответствующую функцию. Расширение функционала калькулятора производится динамически (на рантайме, путем анализа файлов в директории plugins), без дополнительной рекомпиляции.
```
./calc.exe
./plugins/funcsin.dll
./plugins/funcdeg.dll
```

Что приводит к возможности использовать эти функции сразу же после запуска приложения, делая валидными вычисления: ```2^4 + sin(90)```

Каждый плагин содержит ровно одну функцию. Имя файла dll не должно участвовать на определение самой функции и ее имени. 

Необходимо определить и обработать исключительные случаи (некорректная dll, нет никаких dll, какая-то из функций бросает исключение, например, ```ln(-100)``` )

Текущее расположение:
```
->include
└─>.h файлы
-> src
└─>.cpp файлы
->plugins
└─>func(operation).dll
->plugins_view
└─>реализация .dll библиотек
->CMakeLists.txt
```

О калькуляторе:
1) Калькулятор поддерживает базовые операции ```+ - / *```
2) Остальные функции (```sin() cos() ln() ^```) реализованы через ```.dll```
3) Калькулятор во время запуска проверяет наличие библиотек и подгружает их, попутно проверяя их валидность
4) отсутствие библиотек не приводит к выбросу исключения, калькулятор продолжает работать, но с базовыми операциями

Архитектура:
1) Интерфейс ```IOperator``` - шаблон, которому должна соответсвовать любая операция, будь то встроенная или базовая
2) Класс ```Register``` - фабрика и хранилище всех операций
3) Адаптер ```PluginAdapter``` - оборачивает C функции в C++ объекты для совместимости с ```IOperator```

Цикл жизни:
1) Запуск
2) Ввод и парсинг
3) Построение приоритетности
4) Построение дерева
5) Вычисление

Отказоустойчивость:
1) Проверка на наличие библиотек
2) Проверка на валидность библиотеки
3) Проверки на валидность самой математической операции
4) проверка на скобки и унарный минус
5) игнорирование пробелов

Как запускать:

Для ```IDE/IDLE``` - ```File -> Open -> Folder/CMake```, найти и открыть ```CMakeLists.txt```

Для ```Shell```:
1) перейти в скачанную папку на один уровень с ```CMakeLists.txt```
2) ```mkdir build```<br>
   ```cd build```
3) ```cmake .. -G "Ninja"```<br>
   ```cmake --build .```
